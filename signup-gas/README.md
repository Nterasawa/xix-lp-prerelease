# XiX 先行受付フォーム — GAS セットアップ手順

## 概要
- LPの先行受付フォームから送信された内容を、Google Apps Script経由で以下に送信します。
  - **店舗通知**: `info@tonchi.works`（返信ボタンは申込者メール宛）
  - **申込者自動返信**: 申込者のメールアドレス（返信先は `terasawa@tonchi.works`）

## 事前準備
- Googleアカウントで `terasawa@tonchi.works` にログインできる状態にしておく（送信元になります）

## 手順（所要約10分）

### 1. スクリプトを作成
1. [script.google.com](https://script.google.com/) に **terasawa@tonchi.works** でログイン
2. **「新しいプロジェクト」** をクリック
3. プロジェクト名を **「XiX先行受付フォーム」** に変更（左上の「無題のプロジェクト」を書き換え）
4. デフォルトの `Code.gs` の中身を全選択して削除し、このフォルダの **`Code.gs`** の内容を貼り付け
5. `Cmd+S` で保存

### 2. Web App としてデプロイ
1. 右上の **「デプロイ」** → **「新しいデプロイ」** をクリック
2. 歯車アイコン（種類の選択）→ **「ウェブアプリ」** を選択
3. 以下のように設定:
   - **説明**: `XiX 先行受付 v1`
   - **次のユーザーとして実行**: `自分 (terasawa@tonchi.works)`
   - **アクセスできるユーザー**: `全員` ⚠️ 重要
4. **「デプロイ」** をクリック
5. 初回は **「アクセスを承認」** → Google のセキュリティ警告が出たら:
   - 「詳細」→ 「XiX先行受付フォーム（安全ではないページ）に移動」
   - Gmailからのメール送信権限を許可

### 3. URLをコピー
デプロイ完了画面に表示される **「ウェブアプリ URL」** をコピーします。
URLは以下のような形式です:
```
https://script.google.com/macros/s/AKfycbx................./exec
```

### 4. 私 (Claude) に URL を教えてください
そのURLをチャットで送っていただければ、LP側（`js/signup.js`）に設定してcommit+pushします。

## テスト方法（デプロイ後）
1. LPの先行受付フォームにテスト入力→送信
2. 指定したメールアドレスに2通届くことを確認:
   - 店舗側: `info@tonchi.works` に「【XiX 先行受付】新規申込」
   - 申込者側: 入力したメールアドレスに「【XiX】先行受付お申込みを受け付けました」

## トラブルシュート

| 症状 | 原因・対処 |
|---|---|
| 送信しても何も起きない | GAS URL が正しく設定されていない / ブラウザのコンソール (F12) でエラー確認 |
| 権限エラー | 「アクセスを承認」をやり直す / デプロイ設定の「アクセスできるユーザー: 全員」を確認 |
| メールが届かない | Gmail の送信済みフォルダを確認 / 迷惑メール振り分けを確認 |
| CORSエラー | GAS URL が `/exec` で終わっているか確認 / 再デプロイ |

## 再デプロイについて
コード変更後は **「デプロイ」→「デプロイを管理」→ 既存のデプロイを編集 → バージョン「新しいバージョン」** を選んでデプロイ。URLは変わりません。

**新規デプロイするとURLが変わる**ので注意（既存デプロイを編集してください）。

## 任意: スプレッドシート記録を有効化したい場合
`Code.gs` の末尾にコメントアウトされた `_appendToSheet` 関数があります。スプレッドシート ID を設定して `doPost` 内でコメントを外せば、受付内容が自動記録されます。
